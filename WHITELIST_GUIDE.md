# 🔒 蓝牙鼠标白名单功能使用指南

## ✨ 功能概述

已成功实现 MAC 地址白名单功能，可以精确控制哪些设备可以连接到你的手机蓝牙鼠标。

### 主要特性
- ✅ **MAC 地址过滤**：只允许白名单中的设备连接
- ✅ **自动拒绝**：非白名单设备（如手环）会被自动拒绝
- ✅ **扫描周围设备**：一键扫描附近的蓝牙设备
- ✅ **快速添加**：从扫描列表中直接添加到白名单
- ✅ **实时管理**：随时添加/移除白名单设备
- ✅ **详细日志**：所有连接尝试都会记录到 Logcat

## 📱 使用方法

### 1. 进入白名单设置

1. 打开 TVControl 应用
2. 进入"🔵 蓝牙鼠标"界面
3. 点击状态卡片上的 **"🔒 白名单"** 按钮

### 2. 查看当前白名单

对话框顶部会显示：
- 当前白名单中的设备数量
- 每个设备的 MAC 地址
- 移除按钮（❌）

**如果白名单为空**：表示允许所有设备连接

### 3. 扫描周围设备

1. 点击 **"🔍 扫描周围设备"** 按钮
2. 等待 10 秒（自动停止）或点击 **"⏹ 停止扫描"**
3. 扫描结果会显示在列表中，包括：
   - 设备名称
   - MAC 地址
   - 信号强度（极强/强/中等/弱/极弱）

### 4. 添加设备到白名单

#### 方法一：从扫描列表添加
1. 在扫描结果中找到目标设备（你的电脑）
2. 点击右侧的 **"添加"** 按钮
3. 设备会立即加入白名单，背景变为绿色

#### 方法二：使用当前连接的设备
1. 确保电脑已经连接
2. 点击 **"➕ 使用当前连接的设备 (XX:XX:XX...)"** 按钮
3. 电脑的 MAC 地址会自动添加到白名单

#### 方法三：手动输入（已移除，建议用扫描）

### 5. 移除设备

在白名单列表中，点击设备右侧的 ❌ 图标即可移除。

### 6. 清除整个白名单

点击 **"🗑 清除白名单（允许所有设备）"** 按钮，恢复为允许所有设备连接。

## 🎯 典型使用场景

### 场景一：首次设置（防止手环连接）

1. **初次使用**（白名单为空）
   - 关闭手环蓝牙（临时）
   - 进入蓝牙鼠标界面
   - 电脑搜索并配对 "TVControl Mouse"
   - 配对成功，UI 显示电脑 MAC 地址

2. **设置白名单**
   - 点击"🔒 白名单"按钮
   - 点击"➕ 使用当前连接的设备"
   - 关闭对话框

3. **完成**
   - 重新打开手环蓝牙
   - 手环尝试连接时会被自动拒绝
   - Logcat 显示：`⚠️ 拒绝未授权设备连接`

### 场景二：多电脑切换

如果需要连接多台电脑：

1. 依次在每台电脑上配对
2. 每次配对后，点击"使用当前连接的设备"添加到白名单
3. 或者扫描后从列表中添加所有需要连接的电脑

### 场景三：临时允许新设备

1. 清除白名单（允许所有设备）
2. 新设备配对
3. 重新设置白名单，包含新旧设备

## 🔧 技术细节

### MAC 地址格式

系统会自动标准化 MAC 地址格式：
- 输入：`AA:BB:CC:DD:EE:FF` 或 `aabbccddeeff`
- 存储：`AABBCCDDEEFF`（大写，无冒号）
- 比较：不区分大小写和格式

### 工作原理

```kotlin
// 1. 设备尝试连接
onConnectionStateChange(device, status, CONNECTED) {
    // 2. 检查 MAC 地址
    if (!isMacAllowed(device.address)) {
        // 3. 拒绝连接
        gattServer.cancelConnection(device)
        return
    }
    // 4. 允许连接
    connectedDevice = device
}
```

## 🐛 常见问题

### Q1: 为什么扫描不到我的电脑？

**可能原因：**
1. 电脑蓝牙未开启
2. 电脑不在可发现模式
3. 距离太远（>10米）
4. 电脑已经在配对状态

**解决方法：**
- 确保电脑蓝牙开启且可被发现
- 靠近手机（1-2米内）
- 取消电脑的现有配对，重新设置为可发现

### Q2: 设置白名单后无法连接？

**检查清单：**
1. MAC 地址是否正确
2. 电脑蓝牙是否开启
3. 是否需要重新配对

**解决步骤：**
1. 清除白名单
2. 重新配对电脑
3. 再次添加 MAC 地址到白名单

### Q3: 白名单会保存吗？

**当前实现：**
- ❌ 仅保存在内存中
- ❌ 应用重启后白名单会清空

**改进建议：**
可以使用 SharedPreferences 持久化保存：
```kotlin
// 保存
prefs.edit().putStringSet("whitelist", allowedMacAddresses).apply()

// 读取
val saved = prefs.getStringSet("whitelist", emptySet()) ?: emptySet()
setAllowedMacAddresses(saved)
```

### Q4: 可以同时连接多个设备吗？

**当前实现：**
- ❌ BLE GATT 服务器只记录最后一个连接
- ✅ 但白名单可以包含多个 MAC 地址
- ✅ 不同时间可以连接不同设备

### Q5: 手环还是能连接怎么办？

**排查步骤：**
1. 确认手环的 MAC 地址不在白名单中
2. 查看 Logcat 是否有拒绝连接的日志
3. 可能是之前的经典蓝牙连接（不是 BLE HID）

**解决：**
- 在手环 App 中断开与手机的连接
- 或在手机蓝牙设置中取消与手环的配对

## 💡 最佳实践

### 1. 首次配置流程
```
1. 清除所有旧配对
2. 暂时关闭附近其他蓝牙设备
3. 配对目标电脑
4. 添加电脑 MAC 到白名单
5. 重新开启其他蓝牙设备
```

### 2. 定期检查
- 偶尔查看 Logcat，确认没有异常连接
- 清理不再使用的白名单条目

### 3. 备份 MAC 地址
记录电脑的 MAC 地址，以便重装系统后快速配置：
```
Windows: ipconfig /all → 蓝牙物理地址
macOS: 系统报告 → 蓝牙 → 地址
```
